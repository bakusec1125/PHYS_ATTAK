# PHYSCK Matrix Framework

### The Open-Source Matrix for Physical Security Tactics, Techniques, and Common Knowledge.

PHYSCK Matrix is a globally-accessible knowledge base of physical adversary tactics and techniques based on real-world observations. It provides a standardized, enumerated language for physical security practitioners to map threats to specific tools and mitigations.

---

## 🏗 The P-T-A Logic
The framework operates on a three-way relational mapping:
1. **Technique (TTP):** The "What" and "How" of the adversary's action.
2. **Tools & Equipment (T&E):** The kinetic or electronic means required.
3. **Mitigations (M):** The Physical, Technical, or Administrative controls that Deter, Detect, Delay, Deny, or Respond.

## 🛡 Privacy-First Contribution
To protect the anonymity of practitioners and researchers, this project:
- Does **NOT** require user accounts or logging.
- Does **NOT** track IP addresses or browser metadata.
- Collects only necessary technical evidence for vetting.

## 🚀 How to Use the Matrix
- **Gap Analysis:** Identify TTPs relevant to your facility and audit your current Mitigation Registry IDs.
- **Red Teaming:** Use the T&E classes to build realistic adversary profiles for physical penetration tests.
- **Standardization:** Move away from "vague risks" to "enumerated TTP IDs" in board-level reporting.
